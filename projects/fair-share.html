The origin

A spreadsheet that became a product

When my partner and I moved in together, we wanted to make sure money was handled fairly. We found proportional splitting. Each person pays a share of expenses based on their income. We built a little spreadsheet to do the maths. It worked, but pulling up Excel on your phone to split dinner got old fast. I'm a UX designer. I saw the friction and figured I could build something better.

The build

My first vibe code

This was early 2023. ChatGPT had just come out, and I had enough HTML, CSS and JavaScript to be dangerous. So I vibe coded a simple calculator. Ugly, hosted on a default GitHub Pages URL. But it worked. We could pull it up at the shops, split groceries, and move on. That was the whole brief.

The craft

Small details, big difference

Once it was working, the UX designer in me couldn't leave it alone. I added localStorage so it remembered your salaries between visits. Made sure the number keypad came up on mobile instead of the text keyboard. Supported 8 currencies with automatic detection. Added the ability to enter multiple expenses, personalise names, and hide salaries if you're not ready for that conversation yet. None of these are features you'd put on a spec. They're the kind of details you only notice when they're missing.

The growth

From 5 clicks to 3,500

I got a proper domain, set up Google Analytics and Search Console, wrote SEO content, and then mostly forgot about it. It grew on its own. No ads, no marketing, just people searching for a fair way to split bills.
